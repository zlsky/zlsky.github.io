事实上这个过程可以分解为4个步骤,分别是预处理(Prepressing),编译(Compilation),汇编(Assembly)和链接.

预编译

在预编译过程中主要是处理预处理指令,包括以下:

  • 将所有的”#define”删除,并且展开所有的宏定义.
  • 处理所有的条件预编译指令,比如”#if”,”#elif”,”#else”,”#endif”.
  • 处理”#include”预编译指令,将被包含的文件插入到该预编译指令的位置,这是一个递归过程,因为include一个文件里面可以再引入另一个文件.
  • 删除所有的注释,”//“,和”/**/“.
  • 添加行号文件名标识,比如#2”hello.c”2,以便于编译时编译器产生调试用的行号及其它信息.
  • 保留所有的#pragma编译器指令,因为编译器需要使用它们.

编译器一般可以分为6步:扫描,语法分析,语义分析,源代码优化,代码生成和目标代码优化.

词法分析

首先是扫描,扫描的任务很简单,就是简单的进行词法分析,用一种叫有限状态机的算法将源代码字符给分割出来.
经过扫描后应该会有如下的的16个记号

记号 类型
array 标识符
[ 左方括号
index 标识符

= 赋值
( 左圆括号
index 标识符
+ 加号
4 数字
) 右圆括号
* 乘号

2 数字

当然这个东西会有一个叫lex,或是flex的程序实现这个功能,我们只需要做配置就可以了

语法分析

语法分析将对扫描器产生的记录进行语法分析,从而产生语法树(Syntax Tree).

这种抽象语法树的东西也有一个工具,叫着yacc.可以跟据用户配置输出这样的语法树

语义分析

语法分析权是完成了对表达式的表面分析,但不知道这语句是否正确,比如指针与指针相乘是不行的.
编译器只能分析静态语法,当然还有对应的动态语义,只能在运行时才知道是否正确.
静态语义通常会包括声明和类型的匹配,类型的转换.

中间语言生成

现在的编译器有很多优化手段,源码级的优化会在源代码层面中优化,例如我们可以将2+6可以优化一下,因为从字面上就可以看到.所以可能会是这样的.

但是这个优化在语法树上是比较困难的,所以源代码优化器会转成中间代码,它是语法树的顺序表示,其它它已经非常接近目标代码了,但是它一般跟目标机器和运行时环境是无关的.在不同的编译器中有着不同的形式,比较常见的有:三地址码,P-代码.
拿我们最常见的三地址码来说吧:
x=y op z
这是有三个地址码表示的将变量y和z进行op操作,op是算法运算符也可以其它任何应用到y和z的操作.

链接整个过程包括了地址和空间分配,符号决议,重定位等相关操作.
